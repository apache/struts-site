The default configuration (struts-default.xml) configures the Dispatcher Result as the default result, which works well with JavaServer Pages. Any JSP 1.2+ container can work with Struts 2 JSP tags immediately.

Getting Started

Because JSP support occurs through the Dispatcher Result, which is the default result type, you donâ€™t need to specify the type attribute when configuring struts.xml:

<action name="test" class="com.acme.TestAction">
    <result name="success">test-success.jsp</result>
</action>

Then in test-success.jsp:

<%@ taglib prefix="s" uri="/struts-tags" %>

<html>
<head>
    <title>Hello</title>
</head>
<body>

Hello, <s:property value="name"/>

</body>
</html>

Where name is a property on your action. Thatâ€™s it!
